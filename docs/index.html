Cold start: rawenv vs Docker Compose

Same project — PostgreSQL 16 + Redis 7 — everything pre-installed. Time from a fully stopped state to services accepting TCP connections. Median of 3 runs on an Apple M2 (16 GiB, macOS 14, Docker Desktop 4.x).

Metric rawenv docker compose Speedup
Time to first service ready 0.28 s 2.10 s 7.5×
Total time (all services ready) 1.08 s 6.82 s 6.3×
Peak RAM 41 MiB 214 MiB + ~1.8 GiB VM 5.2×
Disk footprint 52 MiB 468 MiB images only 9.0×
Background daemon at idle none persistent Linux VM —

rawenv runs Postgres and Redis as native host processes, so its RAM figure is the whole cost. Docker's container memory excludes the Docker Desktop VM, which reserves roughly 1.5–2 GiB before a single container starts. All numbers reproduce with scripts/benchmark.sh — see the full methodology.

How it works

1 rawenv init

Detect your project stack

2 rawenv add node@22

Download & verify SHA256

3 rawenv up

Start all services

4 rawenv shell

Isolated environment ready

PATH

Installs a single static binary to ~/.rawenv/bin on macOS and Linux (Apple Silicon, Intel, x86_64, arm64). Prefer a manual download? Grab the binary for your platform from the latest release.
